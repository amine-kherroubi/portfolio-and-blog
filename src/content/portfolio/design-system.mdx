# Minimal Design System

A production-ready design system built for scalability, accessibility, and developer experience. Used across 15+ applications serving millions of users.

## The Challenge

Create a unified design language that:

- Maintains consistency across multiple products
- Scales from marketing sites to complex dashboards
- Empowers designers and developers equally
- Prioritizes accessibility and performance

## Design Philosophy

### Core Principles

**1. Constraint as Liberation**
Limited components force thoughtful composition. Every element serves a clear purpose.

**2. Accessibility First**
WCAG 2.1 AA compliance isn't optional—it's the foundation. Every component works with keyboard, screen readers, and assistive technologies.

**3. Performance by Default**
Tree-shakeable components, minimal runtime, zero dependencies. Bundle size measured in KB, not MB.

## System Architecture

### Component Structure

```typescript
// Base component with composition
import { forwardRef, ComponentPropsWithoutRef } from 'react';
import { cva, VariantProps } from 'class-variance-authority';

const button = cva('button-base', {
  variants: {
    variant: {
      primary: 'bg-black text-white',
      secondary: 'bg-white text-black border-2 border-black',
      ghost: 'bg-transparent hover:bg-gray-100',
    },
    size: {
      sm: 'px-4 py-2 text-sm',
      md: 'px-6 py-3 text-base',
      lg: 'px-8 py-4 text-lg',
    },
  },
  defaultVariants: {
    variant: 'primary',
    size: 'md',
  },
});

interface ButtonProps
  extends ComponentPropsWithoutRef<'button'>,
    VariantProps<typeof button> {}

export const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ variant, size, className, ...props }, ref) => {
    return (
      <button
        ref={ref}
        className={button({ variant, size, className })}
        {...props}
      />
    );
  }
);
```

## Accessibility Features

### WCAG 2.1 AA Compliance

✅ **Keyboard Navigation**

- Tab order follows visual flow
- Focus indicators on all interactive elements
- Escape key closes modals and dropdowns

✅ **Screen Reader Support**

- ARIA labels and descriptions
- Live regions for dynamic content
- Semantic HTML structure

✅ **Color Contrast**

- Minimum 4.5:1 for normal text
- 3:1 for large text and UI components
- Tested with color blindness simulators

### Testing Strategy

```typescript
// Automated accessibility tests
import { axe, toHaveNoViolations } from 'jest-axe';
import { render } from '@testing-library/react';

expect.extend(toHaveNoViolations);

describe('Button accessibility', () => {
  it('should have no accessibility violations', async () => {
    const { container } = render(<Button>Click me</Button>);

    expect(results).toHaveNoViolations();
  });
});
```
